# AIで職務経歴書を書く方法——エンジニア向け完全テンプレート【2026年版】

転職活動で一番大変な作業の一つが「職務経歴書」の作成です。  
何を書けばいいか、どう表現すればいいか、書いていて嫌になってきた——そんな経験、ありませんか？

実は、**Claude・ChatGPT などのAIを使えば、職務経歴書作成の時間を70%削減できます。**  
今回は、エンジニアがAIで職務経歴書を書くための実践的な手順とテンプレートを紹介します。

---

## なぜエンジニアの職務経歴書は難しいのか

エンジニアの職務経歴書には、特有の難しさがあります。

1. **技術スタックの羅列になりがち** → 何が「強み」なのか伝わらない
2. **成果を数値化しにくい** → 「○○を開発した」で終わってしまう
3. **専門用語が多すぎて人事に伝わらない** → 技術的すぎる説明
4. **書くべき量の見極めが難しい** → 長すぎても短すぎてもNG

この4つの壁を、AIが解決してくれます。

---

## ステップ1: 自己分析をAIに手伝わせる

まず、以下のプロンプトをClaude/ChatGPTに入力してください。

```
以下の職歴情報をもとに、エンジニアとしての強みと市場価値を分析してください。

【職歴】
・会社名・在職期間（省略可）
・使用技術: Python, Django, AWS等（あなたの技術を入力）
・主な業務: バックエンドAPI開発、インフラ整備等
・チーム規模: 5人
・役割: リードエンジニア

【分析してほしいこと】
1. 市場で評価されやすい経験はどれか
2. 他のエンジニアと差別化できるポイント
3. どの求人に応募すると採用確率が高いか
```

このプロンプトを使うと、自分では気づいていなかった強みを客観的に言語化してもらえます。

## ステップ2: 職務内容をSTAR法でAIに変換させる

エンジニアの「業務内容」を魅力的に見せるには、**STAR法**（Situation・Task・Action・Result）が有効です。

```
以下の業務経験を、採用担当者に伝わりやすいSTAR法で書き直してください。

【元の記述】
「Pythonを使ってAPIを開発した。」

【条件】
・STAR法で構造化する
・成果は具体的な数値で表現する（数値が不明な場合は「約〇〇%改善」と推定）
・読みやすい箇条書き形式
・人事担当者にも技術が伝わるよう専門用語を補足
```

**Before / Afterの例:**

**Before（NG例）:**
> Pythonを使ってAPIを開発した。

**After（AI変換後）:**
> 【状況】既存システムのAPIレスポンス速度が遅く、ユーザー離脱率が高い課題があった  
> 【役割】バックエンドAPIの全面リニューアルをリードエンジニアとして担当  
> 【行動】PythonのFastAPIを採用し、N+1問題の解消とRedisキャッシュを実装  
> 【成果】APIレスポンスを平均800ms → 120ms（約85%改善）に削減。ユーザー離脱率15%低下に貢献

## ステップ3: 技術スタックをAIに整理させる

```
以下の技術スタックを、採用担当者・エンジニア面接官の両方に伝わるよう整理してください。

【使用技術（羅列）】
Python, Django, FastAPI, PostgreSQL, Redis, AWS(EC2/RDS/S3/Lambda), Docker, 
GitHub Actions, Terraform, React, TypeScript

【希望するフォーマット】
・熟練度別（メイン/サブ/基礎知識）に分類
・各技術に「どのくらい使ったか」の補足
・採用担当者へのわかりやすい説明文も追加
```

## ステップ4: 全体をまとめる「完成版プロンプト」

以下の「マスタープロンプト」に自分の情報を埋めると、職務経歴書の骨格が一気に完成します。

```
エンジニアの職務経歴書を作成してください。

【基本情報】
・氏名: （任意）
・経験年数: 5年
・志望業種: SaaS/スタートアップ

【職歴】
1社目（3年）:
- 業種: Eコマース系
- ポジション: バックエンドエンジニア
- 使用技術: Python, Django, MySQL, AWS
- 主な業務: 商品管理APIの開発、決済システムの改善
- 規模: チーム4名

2社目（2年）:
- 業種: BtoB SaaS
- ポジション: シニアエンジニア
- 使用技術: Go, PostgreSQL, GCP, Kubernetes
- 主な業務: マイクロサービス化の推進、CI/CDパイプライン構築
- 規模: チーム8名（自分がリード）

【作成してほしい項目】
1. 職務サマリー（3〜5行）
2. 技術スキル一覧（熟練度別）
3. 職務経歴（各社のSTAR法記述）
4. 自己PR（300字）

【トーン】
・自信があるが謙虚
・数値を積極的に使う
・「〜を行いました」より「〜を実現しました」
```

## エンジニア職務経歴書の「数値化」テクニック

「成果を数値で言えない」という声をよく聞きます。でも、以下のパターンで**ほぼ必ず数値化できます**。

| 業務内容 | 数値化の例 |
|---------|-----------|
| バグ修正 | 「バグ件数を月30件 → 5件に削減」|
| 開発スピード改善 | 「リリース頻度を週1回 → 毎日に向上」|
| インフラ改善 | 「サーバーコストを月40万 → 25万に削減（37%削減）」|
| チームビルディング | 「新人エンジニア3名のオンボーディングを担当、3ヶ月で独立稼働を達成」|
| 顧客対応削減 | 「問い合わせ対応自動化により、CS工数を週10時間削減」|

**数値が不明なときのプロンプト:**

```
この業務の成果を数値化したいのですが、正確な数値がわかりません。
以下の情報から、合理的な推定値と表現方法を提案してください。

【業務】キャッシュ実装によるAPIの高速化
【状況】以前のレスポンスが遅いと言われていた。導入後は速いと言われるようになった。
【推定方法】一般的なキャッシュ導入の効果範囲で推定してください。
```

## よくある失敗と修正プロンプト

### 失敗1: 技術的すぎて人事に伝わらない

```
以下の技術的な説明を、非エンジニアの採用担当者にも伝わるよう書き換えてください。
難しい用語は()内に簡単な補足を加えてください。

【元の文】
「KubernetesによるPodの自動スケーリング設定を実装し、トラフィックスパイク時のサービス安定性を向上させた。」
```

### 失敗2: 「〜しました」の受け身表現ばかり

```
以下の職務経歴書の文体を、主体性・リーダーシップが伝わるよう書き換えてください。
受け身表現を能動的な表現に変換してください。

【元の文章】
「新機能の開発を行いました。テストも実施しました。」
```

### 失敗3: 何でもやった感が出ない

```
以下の職歴から、「専門性の高いエンジニア」として印象付けられるよう、
強みを絞り込んで再構成してください。
1〜2つの専門領域に集中した職務経歴書にしてください。
```

## AIで作った職務経歴書を仕上げる3つのチェック

AIが作った文章は、最後に自分でチェックしましょう。

1. **事実確認**: 数値・技術名・在職期間に誤りはないか
2. **一人称の統一**: 「私」か「（省略）」かを統一する
3. **声に出して読む**: 不自然な日本語を修正する
