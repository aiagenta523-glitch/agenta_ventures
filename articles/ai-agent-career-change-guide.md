エンジニアとして転職活動をしたことがある人なら、こんな経験があるはずです。

- 職務経歴書を書くのに数時間かかる

正直、転職活動は「技術力」よりも「転職の戦略・準備」で結果が変わります。そしてこの準備こそが、AIが最も得意とする領域です。

## 1. 職務経歴書の作成：AIが最も得意な領域

### 従来の問題点

職務経歴書の作成で多くのエンジニアが詰まるのが「自分の経験を言語化すること」です。

「Reactでフロントエンドを作りました」
「APIの設計・実装をしました」

技術的には正しいですが、採用担当者が見たいのは「**あなたが何を解決したか**」です。

### AIを使った職務経歴書の書き方

以下のように、具体的な情報を箇条書きでAIに渡します：

```
以下の経験から職務経歴書の担当業務・成果欄を作成してください：

- React + TypeScriptでECサイトのフロントエンドを担当（1年半）
- ユーザー数：月間5万人
- 主要課題：ページ読み込み速度が遅い（3.5秒）
- 対応：画像の遅延読み込み・Reactのメモ化・バンドルサイズ削減
- 成果：読み込み速度1.2秒に改善（65%改善）
- その他：コードレビュー、新人メンター（3名）
```

これを渡すだけで、こんな文章が生成されます：

> 月間5万人が利用するECサイトのフロントエンド開発に従事。パフォーマンス改善施策として画像の遅延読み込み・Reactメモ化・バンドルサイズ最適化を実施し、ページ読み込み速度を3.5秒から1.2秒へと65%改善。3名の新人エンジニアのメンタリングも担当。

**ポイント**: 数字を入れるだけで説得力が格段に上がります。

## 2. 面接対策：弱点を先に潰す

### AIによる模擬面接

面接官役をAIにやってもらう方法が、最も手軽で効果的です。

```
あなたは厳格な技術面接官です。
私はReactエンジニア（経験3年）で○○社の面接を受けます。
以下について、実際の面接のように質問してください：
- パフォーマンス最適化の経験
- チームでの技術的意思決定
- 失敗談とその学び
```

重要なのは「厳格に」という指示です。優しい質問ばかりになると意味がありません。

### よく聞かれるが準備できていない質問

AIに以下を聞くと、盲点を発見できます：

```
Reactエンジニアが中途採用面接で聞かれる「意地悪な質問」トップ10を教えてください
```

実際によく聞かれる難問：
- 「Reactを使わなくていい場面はどんな時ですか？」
- 「チームメンバーと技術選定で意見が割れたときどうしますか？」
- 「レガシーコードのリファクタリングとスケジュールをどう両立しますか？」

これらを事前に考えておくだけで、面接本番の余裕が全然違います。

## 3. 自分の市場価値を調べる

### AIに年収相場を聞く方法

```
以下のスキルセットを持つエンジニアの2026年の市場年収相場を教えてください：

スキル：React/TypeScript/Node.js/AWS（3年経験）
現職年収：450万円
業種：Webサービス系スタートアップ
居住地：東京
```

ただし、AIの知識には限界があります。転職サイトの実際の求人と組み合わせることが重要です。

### よりリアルな市場把握のためのプロンプト

```
○○職種で転職を検討しています。以下の観点で市場環境を分析してください：
1. 現在の需要が高い理由・低い理由
2. 年収アップが見込める企業タイプ（スタートアップ/大手/外資）
3. 同職種で差がつくスキルセット
4. 2〜3年後の需要予測
```

## 4. 年収交渉：最も苦手とするエンジニアが多い場面

多くのエンジニアが「技術的には自信があるが、交渉事が苦手」という傾向があります。

### 交渉シナリオをAIと事前練習する

```
私はエンジニアで、内定をもらいました。
現在：450万円 → 希望：550万円
会社から「450万円でどうですか」と言われた想定で、
私が年収交渉する練習をしてください。
あなたは採用担当者役をやってください。
```

実際に声に出して練習するか、文章で回答を考えることで、本番の緊張が和らぎます。

### 年収交渉で使える具体的なフレーズ

AIに「年収交渉で効果的なフレーズ」を聞くと、こんな回答が得られます：

- 「御社でのキャリアを長期的に考えており、まず希望金額をお伝えしたいのですが...」
- 「現職での実績として○○があり、同等の水準を期待していますが...」
- 「複数社から内定をいただいていますが、御社を最優先に考えています」
